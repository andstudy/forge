 = 지식창출 프로세스의 적. 조직내 침묵 =

 * 스크럼은 자기조직적이다.
  * 프로젝트를 완료한 사람을 새로운 프로젝트에 투입하여 성공한 프로세스를 전파.
  * 구성원들 각자가 스스로 문제를 찾아내고 스스로 해결한다.
  * 창의적인 결과를 만들어내며 서로서로 배우는 분위기로 결국 모두가 발전.

 == SECI Model (지식창출 프로세스) ==
  * Nonaka Ikujiro 가 제안한 지식창출 프로세스 모델.
attachment:secimodel.jpg

  * 조직에서 지식이 창출되는 프로세스를 설명한 모델
  * "앞으로는 각 개별 구성원들이 가지고 있는 지식을 융합시켜 새로운 지식을 끊임없이 창출해내는 조직체계를 만들어내는 기업만이 성공할것"

  * Socialization 사회화 (암묵지 -> 암묵지)
   * 경험 공유를 통해 다시 암묵지를 창출
   * {{{#blue "내가 UnitTest 사용해보니까 이렇게 좋던데? 너도 한번 해보는건 어때?"}}}

  * Externalization 표출화 (암묵지 -> 형식지)
   * 개인의 암묵지를 개념과 언어의 형태로 표현
   * 전체 지식창출 프로세스에서 매우 중요한 역할 담당
   * {{{#blue "지난 몇달간 돌렸던 UnitTest 결과와 UnitTest 활용방안에 대한 세미나를 시작하겠습니다"}}}

  * Combination 종합화 (형식지 -> 형식지)
   * 언어와 숫자 같은 상징을 통해 다양한 개념을 종합하고 체계화
   * {{{#blue "UnitTest 튜토리얼을 참고하여 앞으로 모든파트에서 Unit Test 를 적용하도록 합시다"}}}

  * Internalization 내면화 (형식지 -> 암묵지)
   * 종합화를 통해 창출된 지식이 개인의 암묵지로 변환
   * {{{#blue "UnitTest 라는게 뭔지도 몰랐는데 해보니까 정말 좋군. 그렇다면 코드를 작성하기전에 Test 를 먼저 작성하는건 어떨까?"}}}

  * Externalization 이 지식창출 프로세스에서 가장 중요한 역할담당.
  * 침묵현상이 있는 조직은 이 단계에서 더 진행하지 못함.

 == 조직내 침묵 현상 (Organizational Silence) ==
  * 조직내 침묵 현상 원문
attachment:organizational_silence.pdf

  * 리더혼자 이야기하고 구성원은 조용히 듣고만 있는 현상.
  * 침묵의 폐해가 상당히 심함.
  * 구성원간 아이디어 교류를 차단하여. 협력을 통한 창조적 결과가 나오지못함.
  * 자기조직적인 구조가 될수없음.

  * 침묵현상이 주는 폐해
   1. 조직 내에 협력을 통한 창의성이 발휘되기 어렵다.
   2. 리더의 계획이나 의도가 부하에게 명확히 전달되지 않는다.
   3. 구성원들의 냉소주의 팽배

  * 침묵의 이유
   1. 기분나빠서 말 안하고 만다.
   2. 소귀에 경 읽어봤자 입만 아파.
   3. 소신있게 말했다가 왕따되면 피곤하다.
   4. 괜히 틀리게 말했다가 부정적인 평가를 받는다.
   5. 윗사람에 대한 복종이 미덕이다.

  * 해결책.
   * 리더가 먼저 변해야 한다.
    1. 자기 과신에서 벗어나라.
    2. 커뮤니케이션 하는 법을 배워라
    3. 사실에 근거한 평가를 해야한다.
    4. 상징적인 언행도 중요하다.
    5. 존칭을 써라.
   * 부하들의 변화도 필요하다
    1. 지혜롭게 말하는 법을 배워라
    2. 주인의식을 가져라

  * 역지사지의 정신.

  * 스크럼의 강력한 장점인 자기조직적인 분위기를 만드려면 먼저 조직의 분위기를 만들어야 하지 않을까?

  * 자기 지식을 나누어주지 않는 침묵의 조직에서는 스크럼을 어떻게 적용할수있을까?
