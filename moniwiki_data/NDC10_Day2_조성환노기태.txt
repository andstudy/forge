== 조성환(네오플) - 서비스의 조삼모사 ==
 * 좋은 강연 감사합니다. 혹시 숫자를 수정 혹은 빼고 싶다면 비밀댓글 등으로 알려주세요.
 * 팀 이름이 무려 '네오플 조종실' 영어로는 command center. 나만 모르고 있었다니.
 * 사례 공유 : 해킹 당한 고객 항의하는 비디오(실제인듯)
 * 이유 : 고객이 게임회사를 자신들의 '적'으로 오해하고 있다. 사실 회사는 도의적 책임만 있다.
  * 집에 도둑 들었다고 해서 정부가 보상하진 않는다. 해킹, 보안은 개인의 몫이었다.
 * 2007 년 겨울 : 1일 1400명, 1일 2100건 이메일
 * 해킹 후 돌아온 사람 분석
  * 해킹 당한 직후 10% 미만, 3개월 : 30% 미만, 6개월 : 60% 미만
  * 게다가 결제율, 결제비용도 평균 2만원에서 평균 6천원으로 낮아진다.
 * 복구해 주자 -> 하지만 수동으로 할 방법밖에 없다.
 * 빠르게, 편하게 신고 즉시 전면 복구 가능하게 한다면?
  * 4개월동안 실무자 설득, 2개월동안 이사님 설득.
 * 프로세스 비교
|| 이전 방식 || 개선된 방식 ||
|| 해킹신고 || 해킹신고 ||
|| 해킹 진위여부 확인 || 선복구 {OK} ||
|| 로그 수색 || 해킹 진위여부 확인 ||
|| 가해자 계정 정지 || 가해자 계정 정지 ||
|| 일부 아이템 회수 및 복구(대기자 30일. 아이템 중 30% 복구) || 아이템 회수(대기자 1일, 복구율 100%) ||
 * 선복구 후 해킹 후 돌아온 사람 분석
  * 3개월 90% 정도, 6개월 85% 정도(일반 유저 이탈율보다 약간 높다고)
 * ARPU(Average revenue per user) 유지 : 2.4만원
 * 1일 400명, 1일 600건 이메일로 줄었다.
 * 남은 인원을 로그 분석 같은 업무로 돌릴 수 있었다.
=== 어뷰징 양산 및 대책 ===
 * 어뷰징이 늘고, 선복구가 되니까 보안이 허술해 져서 막 털리기 시작함
 * 어뷰저와 재해킹 유저의 공통점 : item 거래를 많이 하더라
 * 선복구 계정의 경우 해킹 보호의 명목으로 "아이템 30일 거리 정지" 시킴
 * 대신 보안 시스템 설치하면 혜택을 더 줬다
  * 어뷰징 비율 12% -> 0.5% 로 줄었다.
 * 재해킹 비율 30% -> 6%, 보안 설치 비율 15% -> 40%
 * 마치 소잃고 외양간 고쳤는데 소가 돌아온 것 같았다.
 * 선복구 수혜자 : 180000명(2008년 여름부터)
 * 수혜자가 결제한 금액 : 4,320,000,000(43억), 특허도 냈다고
=== Q/A ===
 * 나는 50만원 결제했는데 복구 못 받았다.
  * 문제가 있을 경우 로그,IP 조사해 환불해 준다(이 경우는 선복구가 아닌 듯)
  * 다만, 로그는 이전 3개월까지만 저장 -> 그 이후 복구 신청하면 데이터가 남아있지 않아서 어렵다.
 * 선복구하면 장물 회수 전까지는 같은 아이템이 2개 있는 셈
  * 현금거래는 돌려주지 않는다. 개인상점은 골드로 돌려준다(성수기 40%, 비수기 60%)
 * 보안팀에서는 기술적으로 막으려고 노력 중
 * 장물 처리 인력 : 15명 내외다.

== 노기태(넥슨) - 데이터를 활용한 게임 운영 및 분석 ==
 * [http://devcatpublications.wordpress.com/2010/05/28/%EB%85%B8%EA%B8%B0%ED%83%9C-%E2%80%9C%EB%8D%B0%EC%9D%B4%ED%84%B0%EB%A5%BC-%ED%99%9C%EC%9A%A9%ED%95%9C-%EA%B2%8C%EC%9E%84-%EC%9A%B4%EC%98%81-%EB%B0%8F-%EB%B6%84%EC%84%9D%E2%80%9D-nexon-developer-conf/ PT 자료]
 * 실시간 로그 데이터 확인
 * 그룹웨어 활용한 로그 데이터 활용
 * 로그가 게임에 영향을 주지 않게 하기 위해 원본용 로그는 단순하게, 뷰어용 로그는 유동적으로 관리한다
 * [http://en.wikipedia.org/wiki/SQL_Server_Integration_Services SQL Server Integration Service. SSIS] 추천. 5분마다 데이터를 동기화한다고.(5분마다 스냅샷 뜬다는 얘기일까)
 * 데이터 보기 : 수정/배포가 용이한 웹으로, IIS AD 인증 이용한 보안
 * 신고전 선제재 (proactive 제재)
  * 해킹 흐름을 본다. 패스워드 탈취 후 아이템 판매까지 4~10분이 걸린다.
  * 반복 데이터를 찾는다.
   * 동일 IP(다중 계정 쓰는 경우) -> 해당 IP 블록(단, 넥슨PC 방인지 확인)
   * 계정도 다 차단(다른 IP 받아와서 하니까)
 * 그룹웨어(TalkCAT) 으로 사람들에게 이상 데이터 흐름 notify
 * SSIS (C#, VB.NET 스크립트 사용 가능) -> TalkCAT 연동
 * 로그 -> 치팅 패턴 분석 -> 실시간 블록
  * 아이템 실시간 복구(대기인원 1400명 -3주후-> 2,3명)
 * 결국 해킹을 막지는 못하더라도 해커를 최대한 귀찮게 하면 다른 게임으로 갈 거다.
 * 데이터 분석이 대세
  * [http://hyeonseop.textcube.com/6 엑셀 Data mining Add-ins] [http://www.sqlserverdatamining.com/ssdm/Home/DataMiningAddinsLaunch/tabid/69/Default.aspx 참고2]
  * 주 단위로 샘플링 (10%)
  * 샘플링한 모든 로그를 분석
 * [http://www.microsoft.com/sqlserver/2005/en/us/analysis-services.aspx SQL Server 2005 Analysis Service] : 데이터 다차원 분석
 * 휴면/활성 유저 분석 (33페이지)
  * 왜 휴면했을까? 비슷한 영향관계 나오면 그게 원인
  * 데이터 구성을 어떻게 하는가가 핵심 기술
 * [http://www.powerpivot.com/download.aspx PowerPivot for Excel 2010] 이란 것도 나왔다. 이제 데이터 마이닝이 참 쉬워질 거다.

== TA 패널 토크 ==
 * 김충효, 김주복, 이승철(넥슨), 이상원, 강성구(엔트리브)
 * TA 라는 단어는 2008 GDC 에서 알게 되었다.
 * 여러 툴에 관심 많았는데 어느새 TA 일을 맡고 있더라.
 * TA 는 초파리처럼 자생한다는 의견이 나왔다.
 * Bluehole Studio 에는 아티스트가 100명이 넘고, TA 도 3명이나 있다고
  * 하지만, 대부분의 회사에서는 TA 뽑기가 너무 어렵다. 그래서 소질 있어 보이는 사람을 훈련시키는 방식을 더 선호한다. 뽑아서 여러 파트를 무사 수행시킨다는 의견도 있었음.
 * TA 뽑을려고 구인란을 만들어 보면 필요스킬이 너무 많아서 더욱 더 겁먹고 지원 안 하는 듯하다. 장기적으로 사람을 뽑고, TA 내에서도 직군 세분화가 필요할지도 모른다.
 * TA 육성방법
  * 자수성가형 TA 는 후배 TA 에게 잘 안 알려주려는 경향이 있다.
  * '소질이 중요하다' VS '누구나 노력하면 할 수 있다'
  * TA 가 하던 일을 비TA 에게 하나씩 넘기는 식으로 교육시키고 있다.
  * 위키 만들고, 메뉴얼을 TA 가 쓰게 한다. 일일보고서 쓰게 하고, 그걸 고참 TA 가 첨삭지도한다
   * 용어를 정확하게 쓰는 게 중요하기 때문
 * 좋은 얘기가 많이 나왔지만, 정리가 어렵네요. 2일째 후기는 여기까지...
